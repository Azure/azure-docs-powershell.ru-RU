# Invoke-AzSqlElasticPoolFailover

## КРАТКИй обзор
Отработка отказа эластичного пула.

## Максимальное

```
Invoke-AzSqlElasticPoolFailover [-ElasticPoolName] <String> [-AsJob] [-PassThru] [-Force]
 [-ServerName] <String> [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## NОПИСАНИЕ
Командлет Invoke-AzSqlElasticPoolFailover отработка отказа эластичного пула. Отработка отказа выполняется для всех баз данных в пуле эластичных БД после выполнения этого командлета.

## ИЛЛЮСТРИРУЮТ

### Пример 1
```powershell
PS C:\> Invoke-AzSqlElasticPoolFailover -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -ElasticPoolName "ElasticPool01"
```

Эта команда будет отработка отказа для пула эластичных БД с именем "ElasticPool01" на сервере с именем "Server01".  Это означает, что переход на другой ресурс выполняется для всех баз данных в пуле эластичных БД с именем "ElasticPool01".
