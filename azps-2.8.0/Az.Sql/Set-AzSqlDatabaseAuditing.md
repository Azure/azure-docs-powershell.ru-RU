## NОПИСАНИЕ
Командлет **Set-AzSqlDatabaseAuditing** изменяет параметры аудита для базы данных SQL Azure.
Чтобы использовать командлет, используйте параметры *ResourceGroupName* , *ServerName* и *DatabaseName* для идентификации базы данных.
Назначение журналов аудита определяется указанием одного из следующих параметров переключателя: BlobStorage, LogAnalytics или EventHub (если ничего не указано, по умолчанию используется значение BlobStorage).
Используйте параметр *State* для включения или отключения политики.
Если в качестве назначения журналов аудита задано хранилище BLOB-объектов, укажите параметр *StorageAccountName* , чтобы определить учетную запись хранения для журналов аудита и параметр *StorageKeyType* , чтобы определить ключи хранения. Вы также можете определить хранение журналов аудита, задав значение параметра *RetentionInDays* , чтобы определить период для журналов аудита.
Если командлет завершился успешно и вы используете параметр *PassThru* , он возвращает объект, описывающий текущие параметры аудита в дополнение к идентификаторам базы данных.
Идентификаторы баз данных включают, но не ограничиваются, **ResourceGroupName** , **ServerName** и **DatabaseName**.

## ИЛЛЮСТРИРУЮТ

### Пример 1: Включение политики аудита хранилища BLOB-объектов для базы данных SQL Azure
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -StorageAccountName "Storage22" -DatabaseName "Database01"
```

### Пример 2: отключение политики аудита хранилища BLOB-объектов для базы данных SQL Azure
```
PS C:\>Set-AzSqlDatabaseAuditing -State Disabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01"
```

### Пример 3: Включение политики аудита хранилища BLOB-объектов для базы данных Azure SQL с помощью учетной записи хранения из другой подписки
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -StorageAccountName "Storage22" -StorageAccountSubscriptionId "7fe3301d-31d3-4668-af5e-211a890ba6e3"
```

### Пример 4: Включение политики аудита хранилища BLOB-объектов для базы данных SQL Azure с расширенной фильтрацией с помощью предиката T-SQL
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -StorageAccountName "Storage22" -DatabaseName "Database01" -PredicateExpression "statement <> 'select 1'"
```

### Пример 5. Удаление параметра расширенной фильтрации из политики аудита хранилища BLOB-объектов в базе данных Azure SQL
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -StorageAccountName "Storage22" -DatabaseName "Database01" -PredicateExpression ""
```

### Пример 6: Включение политики аудита центра событий для базы данных SQL Azure
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -EventHub -EventHubName "EventHubName" -EventHubAuthorizationRuleResourceId "EventHubAuthorizationRuleResourceId"
```

### Пример 7: отключение политики аудита центра событий для базы данных SQL Azure
```
PS C:\>Set-AzSqlDatabaseAuditing -State Disabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -EventHub
```

### Пример 8: Включение политики аудита анализа журналов для базы данных SQL Azure
```
PS C:\>Set-AzSqlDatabaseAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01" -LogAnalytics -WorkspaceResourceId "/subscriptions/4b9e8510-67ab-4e9a-95a9-e2f1e570ea9c/resourceGroups/insights-integration/providers/Microsoft.OperationalInsights/workspaces/viruela2"
```

### Пример 9: отключение политики аудита анализа журналов для базы данных SQL Azure
```
PS C:\>Set-AzSqlDatabaseAuditing -State Disabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01" -LogAnalytics
```

### Пример 10: отключение через конвейер, политика аудита анализа журналов для базы данных Azure SQL
```
PS C:\>Get-AzSqlDatabase -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01" | Set-AzSqlDatabaseAuditing -LogAnalytics -State Disabled
```
