# Reset-AzBatchComputeNode

## SYNOPSIS
Переустановка операционной системы на указанном узле вычислений.

## СИНТАКСИС

### ИД (по умолчанию)
```
Reset-AzBatchComputeNode [-PoolId] <String> [-Id] <String> [-ReimageOption <ComputeNodeReimageOption>]
 -BatchContext <BatchAccountContext> [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

### InputObject
```
Reset-AzBatchComputeNode [[-ComputeNode] <PSComputeNode>] [-ReimageOption <ComputeNodeReimageOption>]
 -BatchContext <BatchAccountContext> [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## ОПИСАНИЕ
**Cmdlet Reset-AzBatchComputeNode** переустановит операционную систему на указанном узле вычислений.

## ПРИМЕРЫ

### Пример 1. Повторное переимяние узла
```
PS C:\>Reset-AzBatchComputeNode -PoolId "MyPool" -Id "tvm-3257026573_2-20150813t200938z" -BatchContext $Context
```

Эта команда переимежает узел вычислений с помощью ИД "tvm-3257026573_2-20150813t200938z" в пуле MyPool.
Используйте Get-AzBatchAccountKey для назначения контекста переменной $Context.

### Пример 2. Повторное объединение всех узлов в пуле
```
PS C:\>Get-AzBatchComputeNode -PoolId "MyPool" -BatchContext $Context | Reset-AzBatchComputeNode -BatchContext $Context
```

Эта команда переимежает каждый узел вычислений в пуле MyPool.

## PARAMETERS

Указывает экземпляр **BatchAccountContext,** который этот cmdlet использует для взаимодействия со службой Batch.
Если для Get-AzBatchAccount BatchAccountContext используется Get-AzBatchAccount, при взаимодействии со службой batchy будет использоваться проверка подлинности Azure Active Directory. Чтобы использовать проверку подлинности общего ключа, используйте Get-AzBatchAccountKey для получения объекта BatchAccountContext с заполненными ключами доступа.
