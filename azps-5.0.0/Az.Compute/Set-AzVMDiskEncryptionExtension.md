# Set-AzVMDiskEncryptionExtension

## КРАТКИй обзор
Включает шифрование на работающей виртуальной машине IaaS в Azure.

## NОПИСАНИЕ
Командлет **Set-AzVMDiskEncryptionExtension** позволяет выполнять шифрование на работающей инфраструктуре в качестве виртуальной машины службы (IaaS) в Azure.  Это обеспечивает шифрование, устанавливая расширение шифрования диска на виртуальной машине. 

Этот командлет требует подтверждения от пользователей. чтобы включить шифрование, требуется перезапуск виртуальной машины.

Перед запуском этого командлета рекомендуется сохранить работу на виртуальной машине.

Linux: параметр **VolumeType** требуется при шифровании виртуальных машин Linux, для которого необходимо установить значение ("OS", "Data" или "ALL"), которое поддерживается дистрибутивом Linux. 

Windows: параметр **VolumeType** можно опустить, в противном случае операция по умолчанию имеет значение ALL; Если параметр VolumeType присутствует для виртуальной машины Windows, для него должно быть задано значение ALL или OS.

## ИЛЛЮСТРИРУЮТ

### Пример 1: Включение шифрования
```
$RGName = "MyResourceGroup"
$VMName = "MyTestVM"
$VaultName= "MyKeyVault"
$KeyVault = Get-AzKeyVault -VaultName $VaultName -ResourceGroupName $RGName
$DiskEncryptionKeyVaultUrl = $KeyVault.VaultUri
$KeyVaultResourceId = $KeyVault.ResourceId
$VolumeType = "All"
Set-AzVMDiskEncryptionExtension -ResourceGroupName $RGName -VMName $VMName -DiskEncryptionKeyVaultUrl $DiskEncryptionKeyVaultUrl -DiskEncryptionKeyVaultId $KeyVaultResourceId -VolumeType $VolumeType
```

В этом примере включается шифрование на виртуальной машине без указания учетных данных Active Directory.

### Пример 2: Включение шифрования с помощью конвейерного ввода
```
$params = New-Object PSObject -Property @{
    ResourceGroupName = "[resource-group-name]"
    VMName = "[vm-name]"
    DiskEncryptionKeyVaultId = "/subscriptions/[subscription-id-guid]/resourceGroups/[resource-group-name]/providers/Microsoft.KeyVault/vaults/[keyvault-name]"
    DiskEncryptionKeyVaultUrl = "https://[keyvault-name].vault.azure.net"
    KeyEncryptionKeyVaultId = "/subscriptions/[subscription-id-guid]/resourceGroups/[resource-group-name]/providers/Microsoft.KeyVault/vaults/[keyvault-name]"
    KeyEncryptionKeyUrl = "https://[keyvault-name].vault.azure.net/keys/[kekname]/[kek-unique-id]"
    VolumeType = "All"
}

$params | Set-AzVmDiskEncryptionExtension
```

Этот пример отправляет параметры с помощью конвейера ввода для включения шифрования на виртуальной машине без указания учетных данных Active Directory.

### Пример 3: Включите шифрование с помощью идентификатора клиента Azure AD и секретного клиента

В этом примере идентификатор клиента Azure AD и секрет клиента используются для включения шифрования на виртуальной машине.

Windows: параметр **VolumeType** можно опустить, в противном случае операция по умолчанию имеет значение ALL; Если параметр VolumeType присутствует для виртуальной машины Windows, для него должно быть задано значение ALL или OS.
