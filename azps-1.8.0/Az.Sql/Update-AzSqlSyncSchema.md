# Update-AzSqlSyncSchema

## КРАТКИй обзор
Обновите схему синхронизации для базы данных участника синхронизации или базы данных концентратора синхронизации.
Она будет получать последнюю схему базы данных из реальной базы данных, а затем использовать ее для обновления схемы, кэшированной с помощью базы данных метаданных синхронизации.
Если задано значение "SyncMemberName", будет обновлена схема базы данных участника; в противном случае будет обновлена схема базы данных основного сервера.

## Максимальное

```
Update-AzSqlSyncSchema [-SyncGroupName] <String> [-SyncMemberName <String>] [-PassThru] [-ServerName] <String>
 [-DatabaseName] <String> [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## NОПИСАНИЕ
Командлет **Update-AzSqlSyncSchema** обновляет схему синхронизации для базы данных участника синхронизации или базы данных концентратора синхронизации.

## ИЛЛЮСТРИРУЮТ

### Пример 1: обновление схемы синхронизации для базы данных-концентратора
```
PS C:\>Update-AzSqlSyncSchema -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "database01" -SyncGroupName "syncGroup01"
```

Эта команда обновляет схему синхронизации для базы данных Hub в группе "Синхронизация" syncGroup01

### Пример 2: обновление схемы синхронизации для базы данных участника
```
PS C:\>Update-AzSqlSyncSchema -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "database01" -SyncGroupName "syncGroup01" -SyncMemberName "syncMember01"
```

Эта команда обновляет схему синхронизации для базы данных пользователей в элементе Sync syncMember01
