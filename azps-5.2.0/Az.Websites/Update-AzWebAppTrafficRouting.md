# Update-AzWebAppTrafficRouting

## SYNOPSIS
Обновив правило маршрутки до слота.

## СИНТАКСИС

```
Update-AzWebAppTrafficRouting -ResourceGroupName <String> -WebAppName <String> -RoutingRule <Hashtable>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## ОПИСАНИЕ
С помощью cmdlet **Update-AzWebAppTrafficRouting** можно обновить конфигурацию правила маршрутирования для слота Azure Web App.

## ПРИМЕРЫ

### Пример 1: обновление правила маршрутки для переноса 15 % производственного трафика в слот Stg
```powershell
PS C:\>Update-AzWebAppTrafficRouting -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
- RoutingRule @{AtionHostName='XXXX.azurewebsites.net';ReroutePercentage=15;Name='Stg'}
```

Эта команда обновляет правило маршрутки, чтобы перенести 15 % производственного трафика в слот Stg.

### Пример 2: обновление правила маршрутки для передачи производственного трафика в диапазоны от 50% до 90 % добавочно.
```powershell
PS C:\>Update-AzWebAppTrafficRouting -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-RoutingRule @{ActionHostName='XXXX.azurewebsites.net';ReroutePercentage=50;ChangeIntervalInMinutes=1;
MinReroutePercentage=50;MaxReroutePercentage=90;Name='Stg';ChangeStep=10}
```

Эта команда обновляет правило маршрутки, чтобы переносить производственный трафик в диапазоны от 50% до 90 % добавочно.
