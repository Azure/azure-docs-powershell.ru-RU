# Set-AzSecurityAssessment

## КРАТКИй обзор
Создание или обновление результатов оценки системы безопасности для ресурса

## Максимальное

### SubscriptionLevelResource (по умолчанию)
```
Set-AzSecurityAssessment -Name <String> [-StatusCode <String>] [-StatusCause <String>]
 [-StatusDescription <String>]
 [-AdditionalData <System.Collections.Generic.Dictionary`2[System.String,System.String]>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdLevelResource
```
Set-AzSecurityAssessment -Name <String> -AssessedResourceId <String> -StatusCode <String>
 [-StatusCause <String>] [-StatusDescription <String>]
 [-AdditionalData <System.Collections.Generic.Dictionary`2[System.String,System.String]>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## NОПИСАНИЕ
Создание или обновление результатов оценки системы безопасности для ресурса можно использовать для изменения состояния существующего результата или добавления дополнительных данных.
можно использовать только для типов оценки "CustomerManaged" и только после создания сопоставленных метаданных оценки.

## ИЛЛЮСТРИРУЮТ

### Пример 1
```powershell
PS C:\> Set-AzSecurityAssessment -Name 4FB6C0A0-1137-42C7-A1C7-4BD37C91DE8D -StatusCode "Unhealthy"
```

Помечает результат подписки как "неработоспособный" для оценки типа "4FB6C0A0-1137-42C7-A1C7-4BD37C91DE8D" — Дополнительные сведения о типе оценки будут обнаружены в типе assessmentMetadata
