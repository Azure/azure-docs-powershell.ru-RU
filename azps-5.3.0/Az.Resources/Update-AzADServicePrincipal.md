# Update-AzADServicePrincipal

## SYNOPSIS
Обновляет существующую главную службу Azure Active Directory.

## СИНТАКСИС

### SpObjectIdWithDisplayNameParameterSet (по умолчанию)
```
Update-AzADServicePrincipal -ObjectId <String> [-DisplayName <String>] [-Homepage <String>]
 [-IdentifierUri <String[]>] [-KeyCredential <KeyCredential[]>] [-PasswordCredential <PasswordCredential[]>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### SpApplicationIdWithDisplayNameParameterSet
```
Update-AzADServicePrincipal -ApplicationId <Guid> [-Homepage <String>] [-IdentifierUri <String[]>]
 [-KeyCredential <KeyCredential[]>] [-PasswordCredential <PasswordCredential[]>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## ОПИСАНИЕ
Обновляет существующую главную службу Azure Active Directory. Чтобы обновить учетные данные, связанные с этим принципом обслуживания, используйте New-AzADSpCredential- Чтобы обновить свойства, связанные с приложением, используйте Update-AzADApplication-

## ПРИМЕРЫ

### Пример 1. Обновление отображаемого имени директора-службы

```powershell
PS C:\> Update-AzADServicePrincipal -ObjectId 784136ca-3ae2-4fdd-a388-89d793e7c780 -DisplayName MyNewDisplayName
```

Отображает имя директора-службы с помощью ид объекта "784136ca-3ae2-4fdd-a388-89d793e7c780" на "MyNewDisplayName".

### Пример 2. Обновление отображаемого имени основной услуги с помощью piping

```powershell
PS C:\> Get-AzADServicePrincipal -ObjectId 784136ca-3ae2-4fdd-a388-89d793e7c780 | Update-AzADServicePrincipal -DisplayName MyNewDisplayName
```

Получает главную службу с ид объекта '784136ca-3ae2-4fdd-a388-89d793e7c780' и каналами к Update-AzADServicePrincipal, чтобы обновить отображаемого имени директора-службы на "MyNewDisplayName".

### Пример 3
