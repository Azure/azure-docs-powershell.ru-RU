# Invoke-AzSqlDatabaseFailover

## КРАТКИй обзор
Отработка отказа базы данных.

## Максимальное

```
Invoke-AzSqlDatabaseFailover [-DatabaseName] <String> [-ReadableSecondary] [-AsJob] [-PassThru] [-Force]
 [-ServerName] <String> [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## NОПИСАНИЕ
Командлет Invoke-AzSqlDatabaseFailover отработка отказа базы данных SQL Azure. Если база данных находится в пуле эластичных БД, эта команда будет отпустить указанную базу данных, не затрагивая другие базы данных в том же пуле эластичных БД.

## ИЛЛЮСТРИРУЮТ

### Пример 1
```powershell
PS C:\> Invoke-AzSqlDatabaseFailover -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01"
```

Эта команда будет отработка отказа основной реплики базы данных с именем "Database01" на сервере с именем "Server01"

### Пример 2
```powershell
PS C:\> Invoke-AzSqlDatabaseFailover -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01" -ReadableSecondary
```

Эта команда отменяет отработку отказа для считывания вторичной реплики базы данных с именем "Database01" на сервере с именем "Server01"
