# Remove-AzureRmPolicySetDefinition

## КРАТКИй обзор
Удаление определения политики.

[!INCLUDE [migrate-to-az-banner](../../includes/migrate-to-az-banner.md)]

## Максимальное

### Параметр имени определения политики задает. По умолчанию
```
Remove-AzureRmPolicySetDefinition -Name <String> [-Force] [-ApiVersion <String>] [-Pre]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### Параметр идентификатора определения политики задает значение.
```
Remove-AzureRmPolicySetDefinition -Id <String> [-Force] [-ApiVersion <String>] [-Pre]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## NОПИСАНИЕ
Командлет **Remove-AzureRmPolicySetDefinition** удаляет определение политики.

## ИЛЛЮСТРИРУЮТ

### Пример 1: Удаление определения политики для идентификатора ресурса
```
PS C:\>$PolicySetDefinition = Get-AzureRmPolicySetDefinition -ResourceId "/subscriptions/mySub/Microsoft.Authorization/policySetDefinitions/myPSSetDefinition"
PS C:\>Remove-AzureRmPolicySetDefinition -Id $PolicySetDefinition.ResourceId -Force
```

Первая команда получает определение политики с помощью командлета Get-AzureRmPolicySetDefinition.
Команда сохраняет его в переменной $PolicySetDefinition.

Вторая команда удаляет определение политики, определяемое свойством **ResourceId** $PolicySetDefinition.
