# Import-AzAutomationRunbook

## КРАТКИй обзор
Импортирует Runbook автоматизации.

## Максимальное

```
Import-AzAutomationRunbook [-Path] <String> [-Description <String>] [-Name <String>] [-Tags <IDictionary>]
 -Type <String> [-LogProgress <Boolean>] [-LogVerbose <Boolean>] [-Published] [-Force]
 [-ResourceGroupName] <String> [-AutomationAccountName] <String> [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## NОПИСАНИЕ
Командлет **Import-AzAutomationRunbook** импортирует Runbook службы автоматизации Azure. Укажите путь к файлу сценария wps_2 (PS1), который нужно импортировать для wps_2 и wps_2 runbooks для рабочих процессов, (graphrunbook) для графических модулей runbooks или (файл. корректировки) для модулей Runbook 2. Для wps_2 Runbook рабочего процесса этот сценарий должен содержать одно определение рабочего процесса wps_2, которое соответствует имени файла.

## ИЛЛЮСТРИРУЮТ

### Пример 1: импорт модуля Runbook из файла
```
PS C:\> $Tags = @{"tag01"="value01"; "tag02"="value02"}
PS C:\> Import-AzAutomationRunbook -Path .\GraphicalRunbook06.graphrunbook -Tags $Tags -ResourceGroup "ResourceGroup01" -AutomationAccountName "AutomationAccount01" -Type GraphicalPowershell
```

Первая команда присваивает переменной $Tags две пары "ключ-значение".
Вторая команда импортирует графический модуль Runbook, именуемый GraphicalRunbook06, в учетную запись автоматизации с именем AutomationAccount01.
Команда также назначает теги, хранящиеся в $Tags.
