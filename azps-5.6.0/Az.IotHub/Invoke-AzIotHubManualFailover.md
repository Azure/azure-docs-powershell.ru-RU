# Invoke-AzIotHubManualFailover

## SYNOPSIS
Вызов отступа для концентратора IoT в геосопозиционном регионе аварийного восстановления.

## СИНТАКСИС

### Набор ресурсов (по умолчанию)
```
Invoke-AzIotHubManualFailover [-ResourceGroupName] <String> [-Name] <String> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObjectSet
```
Invoke-AzIotHubManualFailover [-InputObject] <PSIotHub> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdSet
```
Invoke-AzIotHubManualFailover [-ResourceId] <String> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## ОПИСАНИЕ
Он активирует отбой концентратора IoT на дополнительное место. Это действие приведет к простою решения и потере данных телеметрии. Операция является длительной, и ее можно завершить в течение нескольких минут. Будьте осторожны при его использовании.

## ПРИМЕРЫ

### Пример 1
```powershell
PS C:\> Invoke-AzIotHubManualFailover -ResourceGroupName "myresourcegroup" -Name "myiothub"
```

Инициирование отладки концентратора IoT "myiothub".

### Пример 2
```powershell
PS C:\> Invoke-AzIotHubManualFailover -ResourceGroupName "myresourcegroup" -Name "myiothub" -AsJob
```

Инициирование отладки концентратора IoT "myiothub" в фоновом режиме.
