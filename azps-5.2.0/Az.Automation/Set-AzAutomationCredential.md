# Set-AzAutomationCredential

## SYNOPSIS
Изменяет учетные данные автоматизации.

## СИНТАКСИС

```
Set-AzAutomationCredential [-Name] <String> [-Description <String>] [-Value <PSCredential>]
 [-ResourceGroupName] <String> [-AutomationAccountName] <String> [-DefaultProfile <IAzureContextContainer>]
 [<CommonParameters>]
```

## ОПИСАНИЕ
**Cmdlet Set-AzAutomationCredential** изменяет учетные данные как **объект PSCredential** в автоматизации Azure.

## ПРИМЕРЫ

### Пример 1. Обновление учетных данных
```
PS C:\>$User = "Contoso\DChew"
PS C:\> $Password = ConvertTo-SecureString "Password" -AsPlainText -Force
PS C:\> $Credential = New-Object -TypeName System.Management.Automation.PSCredential -ArgumentList $User, $Password
PS C:\> Set-AzAutomationCredential -AutomationAccountName "Contoso17" -Name "ContosoCredential" -ResourceGroupName "ResourceGroup01" -Value $Credential
```

Первая команда назначает имя пользователя переменной $User.
Вторая команда преобразует пароль в виде обычного текста в безопасную строку с помощью ConvertTo-SecureString командлета.
Эта команда сохраняет объект в переменной $Password.
Третья команда создает учетные данные на основе $User и $Password, а затем сохраняет их в $Credential переменной.
Конечная команда изменяет учетные данные автоматизации с именем ContosoCredential для использования учетных данных в $Credential.
