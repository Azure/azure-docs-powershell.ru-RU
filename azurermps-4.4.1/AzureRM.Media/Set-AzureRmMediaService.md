# Set-AzureRmMediaService

## КРАТКИй обзор
Изменяет указанные свойства существующей службы мультимедиа.

[!INCLUDE [migrate-to-az-banner](../../includes/migrate-to-az-banner.md)]

## Максимальное

```
Set-AzureRmMediaService [-ResourceGroupName] <String> [-AccountName] <String> [-Tags <Hashtable>]
 [-StorageAccounts <PSStorageAccount[]>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## NОПИСАНИЕ
Командлет **Set-AzureRmMediaService** изменяет указанные свойства существующей службы мультимедиа.

## ИЛЛЮСТРИРУЮТ

### Пример 1: изменение существующей службы мультимедиа
```
PS C:\>$Tags = @{"tag1" = "value1"; "tag2" = "value2"}
Set-AzureRmMediaService -ResourceGroupName "ResourceGroup123" -AccountName "MediaService001" -Tags $Tags -StorageAccounts $StorageAccounts
```

В первой команде создается последовательность тегов и сохраняются эти теги в переменной с именем $Tags.

Эта вторая команда обновляет службу мультимедиа с именем MediaService001, которая относится к группе ресурсов с именем ResourceGroup123, с тегами, хранящимися в $Tags переменной.
Команда также использует массив объектов конфигурации хранилища, хранящийся в переменной $StorageAccounts.
