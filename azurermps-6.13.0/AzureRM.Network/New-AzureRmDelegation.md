# New-AzureRmDelegation

## КРАТКИй обзор
Создание делегирования служб.

[!INCLUDE [migrate-to-az-banner](../../includes/migrate-to-az-banner.md)]

## Максимальное

```
New-AzureRmDelegation -Name <String> -ServiceName <String> [-DefaultProfile <IAzureContextContainer>]
 [<CommonParameters>]
```

## NОПИСАНИЕ
Командлет **New-AzureRmDelegation** создает делегирование служб, которое можно добавить в подсеть.

## ИЛЛЮСТРИРУЮТ

### Пример 1
```powershell
PS C:\> $delegation = New-AzureRmDelegation -Name "myDelegation" -ServiceName "Microsoft.Sql/servers"
PS C:\> $vnet = Get-AzureRmVirtualNetwork -Name "myVNet" -ResourceGroupName "myResourceGroup"
PS C:\> $subnet = Get-AzureRmVirtualNetworkSubnetConfig -Name "mySubnet" -VirtualNetwork $vnet
PS C:\> $subnet.Delegations.Add($delegation)
PS C:\> Set-AzureRmVirtualNetwork $vnet
```

Первый командлет создает делегирование, которое можно добавить в подсеть, и сохраняет его в переменной $delegation. Вторая и третья командлеты извлекают подсеть для делегирования. Четвертый командлет добавляет созданное делегирование к нужной подсети, а последний командлет отправляет обновленную конфигурацию на сервер.
