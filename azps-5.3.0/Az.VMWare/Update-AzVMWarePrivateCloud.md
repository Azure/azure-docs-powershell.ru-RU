# Update-AzVMWarePrivateCloud

## SYNOPSIS
Обновление частного облака

## СИНТАКСИС

### UpdateExpanded (по умолчанию)
```
Update-AzVMWarePrivateCloud -Name <String> -ResourceGroupName <String> [-SubscriptionId <String>]
 [-IdentitySource <IIdentitySource[]>] [-Internet <InternetEnum>] [-ManagementClusterSize <Int32>]
 [-Tag <Hashtable>] [-DefaultProfile <PSObject>] [-AsJob] [-NoWait] [-Confirm] [-WhatIf] [<CommonParameters>]
```

### UpdateViaIdentityExpanded
```
Update-AzVMWarePrivateCloud -InputObject <IVMWareIdentity> [-IdentitySource <IIdentitySource[]>]
 [-Internet <InternetEnum>] [-ManagementClusterSize <Int32>] [-Tag <Hashtable>] [-DefaultProfile <PSObject>]
 [-AsJob] [-NoWait] [-Confirm] [-WhatIf] [<CommonParameters>]
```

## ОПИСАНИЕ
Обновление частного облака

## ПРИМЕРЫ

### Пример 1. Обновление размера закрытого облака по имени
```powershell
PS C:\> Update-AzVMWarePrivateCloud -Name azps-test-cloud -ResourceGroupName azps-test-group -ManagementClusterSize 4

Location      Name            Type
--------      ----            ----
australiaeast azps-test-cloud Microsoft.AVS/privateClouds
```

Обновление размера частного облака по имени

### Пример 2. Обновление размера закрытого облака с помощью входного объекта
```powershell
PS C:\> Get-AzVMWarePrivateCloud -ResourceGroupName azps-test-group -Name azps-test-cloud | Update-AzVMWarePrivateCloud -ManagementClusterSize 4

Location      Name            Type
--------      ----            ----
australiaeast azps-test-cloud Microsoft.AVS/privateClouds
```

Обновление размера закрытого облака с помощью объекта ввода

## ПРИМЕЧАНИЯ

ALIASES

СЛОЖНЫЕ СВОЙСТВА ПАРАМЕТРОВ

Чтобы создать описанные ниже параметры, создайте hash table, содержащую нужные свойства. Чтобы получить сведения о hash tables, запустите Get-Help about_Hash_Tables.


IDENTITYSOURCE <IIdentitySource[]>: vCenter Single Sign On Identity Sources
  - `[Alias <String>]`: доменное имя NetBIOS
  - `[BaseGroupDn <String>]`: базовое различается имя группы
  - `[BaseUserDn <String>]`: базовое различается имя для пользователей.
  - `[Domain <String>]`: имя DNS домена
  - `[Name <String>]`: имя источника удостоверений
  - `[Password <String>]`. Пароль пользователя Active Directory с минимальным доступом только для чтения к базовому DN для пользователей и групп.
  - `[PrimaryServer <String>]`: URL-адрес основного сервера
  - `[SecondaryServer <String>]`: URL-адрес дополнительного сервера
  - `[Ssl <SslEnum?>]`: защита связи LDAP с помощью SSL-сертификата (LDAPS)
  - `[Username <String>]`: ИД пользователя Active Directory с минимальным доступом только для чтения к базовому DN для пользователей и групп

INPUTOBJECT <IVMWareIdentity> : Параметр identity
  - `[AuthorizationName <String>]`: Имя авторизации каналов ExpressRoute в частном облаке
  - `[ClusterName <String>]`: название кластера в частном облаке
  - `[HcxEnterpriseSiteName <String>]`: имя корпоративного сайта HCX в закрытом облаке
  - `[Id <String>]`: путь удостоверения ресурса
  - `[Location <String>]`: регион Azure
  - `[PrivateCloudName <String>]`: Имя закрытого облака
  - `[ResourceGroupName <String>]`: Имя группы ресурсов. Имя нечувствительно к делу.
  - `[SubscriptionId <String>]`: ИД целевой подписки.
