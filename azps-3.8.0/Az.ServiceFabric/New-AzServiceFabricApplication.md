# New-AzServiceFabricApplication

## КРАТКИй обзор
Создание нового приложения фабрики служб в указанной группе ресурсов и кластере.

## Максимальное

### SkipAppTypeVersion (по умолчанию)
```
New-AzServiceFabricApplication [-ResourceGroupName] <String> [-ClusterName] <String>
 [-ApplicationTypeName] <String> [-ApplicationTypeVersion] <String> -Name <String>
 [-ApplicationParameter <Hashtable>] [-MinimumNodeCount <Int64>] [-MaximumNodeCount <Int64>] [-Force]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### CreateAppTypeVersion
```
New-AzServiceFabricApplication [-ResourceGroupName] <String> [-ClusterName] <String>
 [-ApplicationTypeName] <String> [-ApplicationTypeVersion] <String> -Name <String>
 [-ApplicationParameter <Hashtable>] -PackageUrl <String> [-MinimumNodeCount <Int64>]
 [-MaximumNodeCount <Int64>] [-Force] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## NОПИСАНИЕ
Этот командлет создает новое приложение фабрики служб в указанной группе ресурсов и кластере. Параметр-PackageUrl можно использовать для создания версии Type, если версия типа уже выходит из нее, но в состоянии "сбой" Этот командлет выдаст запрос о том, требуется ли пользователю повторно создать версию типа. Если он продолжает работу в состоянии "не пройден", команда прекратит процесс и создаст исключение.

## ИЛЛЮСТРИРУЮТ

### Пример 1
```powershell
PS C:\> New-AzServiceFabricApplication -ResourceGroupName "testRG" -ClusterName "testCluster" -ApplicationTypeName "TestAppType" -ApplicationTypeVersion "v1" -Name "testApp" -ApplicationParameter @{key0="value0";key1=$null;key2="value2"}
```

В этом примере создается приложение "testApp" в группе ресурсов "testRG" и кластере "testCluster". Тип приложения "TestAppType" версии "v1" уже должен существовать в кластере, а параметры приложения должны быть заданы в манифесте приложения, в противном случае командлет завершится сбоем.

### Пример 2: укажите-PackageUrl, чтобы создать версию типа приложения перед созданием приложения.
```powershell
PS C:\> New-AzServiceFabricApplication -ResourceGroupName "testRG" -ClusterName "testCluster" -ApplicationTypeName "TestAppType" -ApplicationTypeVersion "v1" -Name "testApp" -PackageUrl "https://sftestapp.blob.core.windows.net/sftestapp/testApp_1.0.sfpkg" -ApplicationParameter @{key0="value0";key1=$null;key2="value2"}
```

В этом примере создается тип приложения "TestAppType" Version "v1" с помощью указанного URL-адреса пакета. После этого она продолжит обычную процедуру для создания приложения. Если версия типа приложения уже закрыта, а состояние подготовки — in "Failed", это предложит решить, требуется ли пользователю повторно создать версию типа.
