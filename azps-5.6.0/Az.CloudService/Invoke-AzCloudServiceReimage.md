# Invoke-AzCloudServiceReimage

## SYNOPSIS
Асинхронная операция reimage переустановит операционную систему для экземпляров ролей веб-ролей или сотрудников.

## СИНТАКСИС

### ReimageExpanded (по умолчанию)
```
Invoke-AzCloudServiceReimage -Name <String> -ResourceGroupName <String> -RoleInstance <String[]>
 [-SubscriptionId <String>] [-DefaultProfile <PSObject>] [-AsJob] [-NoWait] [-PassThru] [-Confirm] [-WhatIf]
 [<CommonParameters>]
```

### ReimageViaIdentityExpanded
```
Invoke-AzCloudServiceReimage -InputObject <ICloudServiceIdentity> -RoleInstance <String[]>
 [-DefaultProfile <PSObject>] [-AsJob] [-NoWait] [-PassThru] [-Confirm] [-WhatIf] [<CommonParameters>]
```

## ОПИСАНИЕ
Асинхронная операция reimage переустановит операционную систему для экземпляров ролей веб-ролей или сотрудников.

## ПРИМЕРЫ

### Пример 1. Повторное экземпляры роли облачной службы
```powershell
PS C:\> $roleInstances = @("ContosoFrontEnd_IN_0", "ContosoBackEnd_IN_1")
PS C:\> Invoke-AzCloudServiceReimage -ResourceGroupName "ContosOrg" -CloudServiceName "ContosoCS" -RoleInstance $roleInstances
```

Эта команда переимежает два экземпляра ролей ContosoFrontEnd_IN_0 и ContosoBackEnd_IN_1 облачной службы ContosoCS, которая принадлежит к группе ресурсов ContosOrg.

### Пример 2. Повторное переимяние всех ролей облачной службы
```powershell
PS C:\> Invoke-AzCloudServiceReimage -ResourceGroupName "ContosOrg" -CloudServiceName "ContosoCS" -RoleInstance "*"
```

Эта команда переимежает все экземпляры ролей облачной службы ContosoCS, которые принадлежат к группе ресурсов ContosOrg.
