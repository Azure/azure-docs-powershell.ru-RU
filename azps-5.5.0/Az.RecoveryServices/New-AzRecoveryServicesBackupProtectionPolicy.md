# New-AzRecoveryServicesBackupProtectionPolicy

## SYNOPSIS
Создает политику защиты от резервного копирования.

## СИНТАКСИС

```
New-AzRecoveryServicesBackupProtectionPolicy [-Name] <String> [-WorkloadType] <WorkloadType>
 [[-BackupManagementType] <BackupManagementType>] [[-RetentionPolicy] <RetentionPolicyBase>]
 [[-SchedulePolicy] <SchedulePolicyBase>] [-VaultId <String>] [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## ОПИСАНИЕ
**Cmdlet New-AzRecoveryServicesBackupProtectionPolicy** создает политику защиты резервного копирования в хранилище.
Политика защиты связана по крайней мере с одной политикой хранения.
Политика хранения определяет срок хранения точки восстановления с помощью резервной копии Azure.
Вы можете использовать Get-AzRecoveryServicesBackupRetentionPolicyObject для получения политики хранения по умолчанию.
Вы также можете использовать Get-AzRecoveryServicesBackupSchedulePolicyObject календарный план, чтобы получить политику расписания по умолчанию.
Объекты **SchedulePolicy** и **RetentionPolicy** используются в качестве входных данных для cmdlet **New-AzRecoveryServicesBackupProtectionPolicy.**
Прежде чем использовать текущий Set-AzRecoveryServicesVaultContext, задате контекст хранилища с помощью Set-AzRecoveryServicesVaultContext.

## ПРИМЕРЫ

### Пример 1. Создание политики защиты от резервного копирования
```powershell
PS C:\> $SchPol = Get-AzRecoveryServicesBackupSchedulePolicyObject -WorkloadType "AzureVM" 
PS C:\> $SchPol.ScheduleRunTimes.Clear()
PS C:\> $Dt = Get-Date
PS C:\> $SchPol.ScheduleRunTimes.Add($Dt.ToUniversalTime())
PS C:\> $RetPol = Get-AzRecoveryServicesBackupRetentionPolicyObject -WorkloadType "AzureVM" 
PS C:\> $RetPol.DailySchedule.DurationCountInDays = 365
PS C:\> New-AzRecoveryServicesBackupProtectionPolicy -Name "NewPolicy" -WorkloadType AzureVM -RetentionPolicy $RetPol -SchedulePolicy $SchPol
```

Первая команда получает базовое **значение SchedulePolicyObject,** а затем сохраняет его в переменной $SchPol.
Вторая команда удаляет все запланированное время запуска из политики расписания в $SchPol.
Третья команда использует Get-Date для получения текущих даты и времени.
Четвертая команда добавляет текущую дату и время в $Dt как запланированное время запуска в политику расписания.
Пятая команда получает базовый объект **RetentionPolicy,** а затем сохраняет его в $RetPol переменной.
Шестая команда задает для политики срока хранения 365 дней.
Конечная команда создает объект **BackupProtectionPolicy** на основе политик расписания и хранения, созданных предыдущими командами.
