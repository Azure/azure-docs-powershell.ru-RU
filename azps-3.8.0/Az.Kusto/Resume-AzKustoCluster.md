# Resume-AzKustoCluster

## КРАТКИй обзор
Возобновление приостановленного кластера Kusto.

## Максимальное

### ByNameAndResourceGroup (по умолчанию)
```
Resume-AzKustoCluster [[-ResourceGroupName] <String>] [-Name] <String> [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByResourceId
```
Resume-AzKustoCluster [-ResourceId] <String> [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByInputObject
```
Resume-AzKustoCluster [-InputObject] <PSKustoCluster> [-PassThru] [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## NОПИСАНИЕ
Возобновление приостановленного кластера Kusto.

## ИЛЛЮСТРИРУЮТ

### Пример 1: возобновление приостановленного кластера Kusto с помощью имени

```
PS C:\> Resume-AzKustoCluster -ResourceGroupName testrg -Name mykustocluster
```

Эта команда возобновляет приостановленный кластер Kusto с именем "mykustocluster", который находится в группе ресурсов "testrg".

### Пример 2: возобновление приостановленного Kustoного кластера с помощью трубопроводов

```
PS C:\> Get-AzKustoCluster -ResourceGroupName testrg -Name mykustocluster | Resume-AzKustoCluster
```

В приведенной выше команде получается кластер Kusto с именем "mykustocluster", обнаруженный в группе ресурсов "testrg", с помощью `Get-AzKustoCluster` командлета, а затем `Resume-AzKustoCluster` продается результат для возобновления.
