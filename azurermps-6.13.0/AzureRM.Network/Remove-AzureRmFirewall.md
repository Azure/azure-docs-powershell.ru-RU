# Remove-AzureRmFirewall

## КРАТКИй обзор
Удаление брандмауэра.

[!INCLUDE [migrate-to-az-banner](../../includes/migrate-to-az-banner.md)]

## Максимальное

```
Remove-AzureRmFirewall -Name <String> -ResourceGroupName <String> [-Force] [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## NОПИСАНИЕ
Командлет **Remove-AzureRmFirewall** удаляет брандмауэр Azure.

## ИЛЛЮСТРИРУЮТ

### 1: создание и удаление брандмауэра
```
New-AzureRmFirewall -Name "azFw" -ResourceGroupName "rgName" -Location centralus 

Remove-AzureRmFirewall -Name "azFw" -ResourceGroupName "rgName"
Confirm
Are you sure you want to remove resource 'azFw'
[Y] Yes  [N] No  [S] Suspend  [?] Help (default is "Y"): y
```

В этом примере создается брандмауэр, а затем он удаляется. Чтобы отключить вывод запроса при удалении брандмауэра, используйте флаг-force.

### 2: выделяйте брандмауэр и удалите брандмауэр
```
$firewall=Get-AzureRmFirewall -ResourceGroupName rgName -Name azFw
$firewall.Deallocate()
Remove-AzureRmFirewall -ResourceGroupName rgName -Name azFw
Confirm
Are you sure you want to remove resource 'azFw'
[Y] Yes  [N] No  [S] Suspend  [?] Help (default is "Y"): y
```

В этом примере извлекается брандмауэр, выделяются брандмауэром, а затем удаляется брандмауэр. Команда "освободить" удаляет запущенную службу, но сохраняет конфигурацию брандмауэра. Если пользователь хочет снова запустить службу, необходимо вызвать метод выделения на брандмауэре.
Чтобы отключить вывод запроса при удалении брандмауэра, используйте флаг-force.
