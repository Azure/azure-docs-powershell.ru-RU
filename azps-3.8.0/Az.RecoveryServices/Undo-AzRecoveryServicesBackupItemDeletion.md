# Undo-AzRecoveryServicesBackupItemDeletion

## КРАТКИй обзор
Восстановление элемента с мягким удалением

## Максимальное

```
Undo-AzRecoveryServicesBackupItemDeletion [-Item] <ItemBase> [-Force] [-VaultId <String>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## NОПИСАНИЕ
Командлет Undo-AzRecoveryServicesBackupItemDeletion восстановление элемента с мягким удалением.
Настройте контекст хранилища с помощью командлета Set-AzRecoveryServicesVaultContext, прежде чем использовать текущий командлет.

## ИЛЛЮСТРИРУЮТ

### Пример 1
```powershell
PS C:\> $Cont = Get-AzRecoveryServicesBackupContainer -ContainerType AzureVM
PS C:\> $PI = Get-AzRecoveryServicesBackupItem -Container $Cont[0] -WorkloadType AzureVM 
PS C:\> Disable-AzRecoveryServicesBackupProtection -Item $PI[0] -RemoveRecoveryPoints
PS C:\> $PI = Get-AzRecoveryServicesBackupItem -Container $Cont[0] -WorkloadType AzureVM 
PS C:\> Undo-AzRecoveryServicesBackupItemDeletion -Item $PI[0]
```

Первая команда получает массив контейнеров резервного копирования, а затем сохраняет их в массиве $Cont.
Вторая команда возвращает элемент резервного копирования, соответствующий первому элементу контейнера, и сохраняет его в переменной $PI.
Третья команда отключает защиту от резервного копирования для элемента в $PI \[ 0 \] и помещает элемент в состояние softdeleted.
Четвертая команда нового элемента, который находится в состоянии softdeleted.
Последняя команда восsoftdeleted ВМ.
