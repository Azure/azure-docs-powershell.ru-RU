# New-AzSupportTicket

## SYNOPSIS
Создается билет в службу поддержки.

## СИНТАКСИС

### CreateSupportTicketWithContactDetailParameterSet (по умолчанию)

 -Severity <Severity> -CustomerFirstName <String> -CustomerLastName <String>
 -PreferredContactMethod <ContactMethod> -CustomerPrimaryEmailAddress <String>
 [-AdditionalEmailAddress <String[]>] [-CustomerPhoneNumber <String>] -CustomerPreferredTimeZone <String>
 -CustomerCountry <String> -CustomerPreferredSupportLanguage <String> [-ProblemStartTime <DateTime>]
 [-CSPHomeTenantId <String>] [-Require24X7Response] [-AsJob] [-DefaultProfile <IAzureContextContainer>]

## ОПИСАНИЕ
С его можно создать вопрос о выставлении счета, управлении подписками, квоте или технических вопросах. Используйте Get-AzSupportService и Get-AzSupportProblemClassification, чтобы определить службу Azure и соответствующие классификации проблем, для которых вы хотите запросить поддержку. Необходимо указать следующие параметры: 

    • Title
    • Description
    • Severity level
    • ProblemClassificationId
    • CustomerContactDetail (or individual customer contact parameters)

Для создания объекта CustomerContactDetail можно использовать New-AzSupportContactProfileObject-помощника.

Поставщики облачных решений могут создать билет в службу поддержки для подписок клиента, войдя в клиент клиента и указав его номер домашнего клиента с помощью параметра *CSPHomeTenantId.*

__Для технических билетов:__

Чтобы указать имя ресурса, укажите ARM ресурса с помощью параметра *TechnicalTicketResourceId.* См. пример ниже. 

__Для билетов на квоту:__

Чтобы запросить увеличение квоты для ядер  **VM Compute,** пакетной базы **данных,** SQL базы данных и хранилища **SQL,** уделять дополнительные сведения *объекту QuotaTicketDetail.* Объект QuotaTicketDetail состоит из 3 свойств, как описано ниже. Для получения подробной документации [щелкните здесь.](https://aka.ms/supportrpquotarequestpayload)

    • QuotaChangeRequestSubType

        This is required for certain quota types when there is a sub type that you are requesting quota increase for. Example: Batch, SQL Database and SQL Data Warehouse have a sub type.

    • QuotaChangeRequestVersion

        This is required and indicates the version of the quota change request payload.

    • QuotaChangeRequests

        This is required and is a list of PSQuotaChangeRequest objects. PSQuotaChangeRequest object has 2 required properties.

        ○ Region

            This is the Azure location or region for which you are requesting quota increase. This is the Location property of Get-AzLocation cmdlet.
        
        ○ Payload

            This is where you specify the new limits for the selected quota type.


Подробные сведения о том, как построить грузку для различных типов квот, можно [найти здесь](https://aka.ms/supportrpquotarequestpayload)
