# New-AzAppServiceEnvironmentInboundServices

## SYNOPSIS
Создает входящие службы для среды служб приложений. Для ASEv2 ILB будет создать частную зону DNS Azure и записи для связи с внутренним IP-адресом. Для ASEv3 она также гарантирует отключение сетевой политики подсети и создание закрытой конечной точки.

## СИНТАКСИС

### SubnetNameParameterSet
```
New-AzAppServiceEnvironmentInboundServices [-ResourceGroupName] <String> [-Name] <String>
 -VirtualNetworkName <String> -SubnetName <String> [-SkipDns] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### SubnetIdParameterSet
```
New-AzAppServiceEnvironmentInboundServices [-ResourceGroupName] <String> [-Name] <String> -SubnetId <String>
 [-SkipDns] [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## ОПИСАНИЕ
С **помощью cmdlet New-AzAppServiceEnvironmentInboundServices** создаются входящие службы для среды служб приложений.

## ПРИМЕРЫ

### Пример 1. Создание личной зоны DNS и записей для ASEv2
```powershell
PS C:\> New-AzAppServiceEnvironmentInboundServices -ResourceGroupName AseResourceGroup -Name AseV2Name
-VirtualNetworkName MyVirtualNetwork -SubnetName InboundSubnet
```

Создание зоны частных DNS и записей для ASEv2

### Пример 2. Создание закрытой конечной точки, личной зоны DNS и записей для ASEv3
```powershell
PS C:\> New-AzAppServiceEnvironmentInboundServices -ResourceGroupName AseResourceGroup -Name AseV2Name
-VirtualNetworkName MyVirtualNetwork -SubnetName InboundSubnet
```

Создание закрытой конечной точки, личной зоны DNS и записей для ASEv3

### Пример 3. Создание закрытой конечной точки для ASEv3
```powershell
PS C:\> New-AzAppServiceEnvironmentInboundServices -ResourceGroupName AseResourceGroup -Name AseV2Name
-VirtualNetworkName MyVirtualNetwork -SubnetName InboundSubnet -SkipDns
```

Создание закрытой конечной точки для ASEv3
