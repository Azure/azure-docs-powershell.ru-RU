# Remove-AzNotificationHubsNamespaceAuthorizationRule

## SYNOPSIS
Удаляет правило авторизации из области имен концентратора уведомлений.

## СИНТАКСИС

```
Remove-AzNotificationHubsNamespaceAuthorizationRule [-ResourceGroup] <String> [-Namespace] <String>
 [-AuthorizationRule] <String> [-Force] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## ОПИСАНИЕ
При этом правило авторизации **Remove-AzNotificationHubsNamespaceAuthorizationRule** удаляет правило авторизации saS из пространства имен концентратора уведомлений.
Правила авторизации управляют доступом к пространству имен.
Для этого нужно создать ссылки ( URIS) на основе различных уровней разрешений.
Уровни разрешений могут быть следующими: 
- Прослушивание
- Отправить
- Управление клиентами направляется на один из этих URL-адресов с учетом соответствующего уровня разрешений.
Например, клиент с разрешением на прослушивание направляется в URI для этого разрешения.
При удалении правила авторизации также удаляется соответствующее разрешение пользователя.

## ПРИМЕРЫ

### Пример 1. Удаление правила авторизации из пространства имен
```
PS C:\>Remove-AzNotificationHubNamespaceAuthorizationRule -Namespace "ContosoNamespace" -ResourceGroup "ContosoNotificationsGroup" -AuthorizationRule "ListenRule"
```

Эта команда удаляет правило авторизации listenRule из пространства имен с именем ContosoNamespace.
При запуске этой команды необходимо указать группу ресурсов, которая назначена области имен.
