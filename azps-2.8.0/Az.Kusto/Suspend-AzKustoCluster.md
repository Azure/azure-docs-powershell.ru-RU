# Suspend-AzKustoCluster

## КРАТКИй обзор
Приостановить существующий кластер Kusto.

## Максимальное

### ByNameAndResourceGroup (по умолчанию)
```
Suspend-AzKustoCluster [-ResourceGroupName] <String> [-Name] <String> [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByResourceId
```
Suspend-AzKustoCluster [-ResourceId] <String> [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByInputObject
```
Suspend-AzKustoCluster [-InputObject] <PSKustoCluster> [-PassThru] [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## NОПИСАНИЕ
Приостановить существующий кластер Kusto.

## ИЛЛЮСТРИРУЮТ

### Пример 1. приостановить существующий кластер Kusto по имени

```
PS C:\> Suspend-AzKustoCluster -ResourceGroupName testrg -Name mykustocluster
```

Приведенная выше команда приостанавливает кластер Kusto с именем "mykustocluster", который находится в группе ресурсов "testrg".

### Пример 2. Приостановка существующего кластера Kusto с помощью трубопроводов

```
PS C:\> Get-AzKustoCluster -ResourceGroupName testrg -Name mykustocluster | Suspend-AzKustoCluster
```

Вышеприведенная команда возвращает кластер Kusto с именем "mykustocluster", обнаруженный в группе ресурсов "testrg", с помощью `Get-AzKustoCluster` командлета, а затем `Suspend-AzKustoCluster` передается результат, чтобы приостановить его.
