# New-AzServiceFabricApplication

## SYNOPSIS
Создайте новое приложение для создания структуры обслуживания в указанной группе ресурсов и кластере.

## СИНТАКСИС

### SkipAppTypeVersion (по умолчанию)
```
New-AzServiceFabricApplication [-ResourceGroupName] <String> [-ClusterName] <String>
 [-ApplicationTypeName] <String> [-ApplicationTypeVersion] <String> -Name <String>
 [-ApplicationParameter <Hashtable>] [-MinimumNodeCount <Int64>] [-MaximumNodeCount <Int64>] [-Force]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### CreateAppTypeVersion
```
New-AzServiceFabricApplication [-ResourceGroupName] <String> [-ClusterName] <String>
 [-ApplicationTypeName] <String> [-ApplicationTypeVersion] <String> -Name <String>
 [-ApplicationParameter <Hashtable>] -PackageUrl <String> [-MinimumNodeCount <Int64>]
 [-MaximumNodeCount <Int64>] [-Force] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## ОПИСАНИЕ
Этот cmdlet создает новое приложение для структуры обслуживания в соответствии с указанной группой ресурсов и кластером. Параметр -PackageUrl можно использовать для создания версии типа. Если тип версии уже выходит, но его состояние имеет состояние "Сбой", будет выдан запрос на повторное создание версии типа. Если процесс продолжится в состоянии "Сбой", команда остановит процесс и выкает исключение.

## ПРИМЕРЫ

### Пример 1
```powershell
PS C:\> New-AzServiceFabricApplication -ResourceGroupName "testRG" -ClusterName "testCluster" -ApplicationTypeName "TestAppType" -ApplicationTypeVersion "v1" -Name "testApp" -ApplicationParameter @{key0="value0";key1=$null;key2="value2"}
```

В этом примере приложение testApp создается в группе ресурсов testRG и кластера testCluster. Тип приложения TestAppType версии v1 уже должен существовать в кластере, а параметры приложения должны быть определены в манифесте приложения. В противном случае сбой cmdlet не удастся.

### Пример 2. Укажите -PackageUrl, чтобы создать версию типа приложения перед его созданием.
```powershell
PS C:\> New-AzServiceFabricApplication -ResourceGroupName "testRG" -ClusterName "testCluster" -ApplicationTypeName "TestAppType" -ApplicationTypeVersion "v1" -Name "testApp" -PackageUrl "https://sftestapp.blob.core.windows.net/sftestapp/testApp_1.0.sfpkg" -ApplicationParameter @{key0="value0";key1=$null;key2="value2"}
```

В этом примере создается тип приложения TestAppType версии v1, используя предоставленный URL-адрес пакета. После этого создание приложения будет продолжаться как обычный процесс. Если версия типа приложения уже выходит из приложения и ее подготовка имеет состояние "Сбой", пользователю будет предложено решить, захочет ли пользователь повторно создать ее.
