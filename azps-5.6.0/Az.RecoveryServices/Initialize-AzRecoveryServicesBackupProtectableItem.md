# Initialize-AzRecoveryServicesBackupProtectableItem

## SYNOPSIS
Эта команда активирует обнаружение незащищенных элементов данного типа рабочей нагрузки в этом контейнере. Если приложение DB не защищено автоматически, используйте эту команду для обнаружения новых DBS при их добавлении и защиты.

## СИНТАКСИС

```
Initialize-AzRecoveryServicesBackupProtectableItem [-Container] <ContainerBase> [-WorkloadType] <WorkloadType>
 [-PassThru] [-VaultId <String>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## ОПИСАНИЕ
этот cmdlet запрашивает конкретные рабочие нагрузки в контейнере. При этом запускается операция, в которой создаются защищенные элементы.

## ПРИМЕРЫ

### Пример 1
```powershell
PS C:\> Initialize-AzRecoveryServicesProtectableItem -Container $Container -WorkloadType "MSSQL"
```

Он выполняет операцию обнаружения новых защищенных элементов.

### Пример 2

Эта команда активирует обнаружение незащищенных элементов данного типа рабочей нагрузки в этом контейнере. (autogenerated)

