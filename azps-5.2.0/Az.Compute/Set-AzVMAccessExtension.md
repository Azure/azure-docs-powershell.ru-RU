# Set-AzVMAccessExtension

## SYNOPSIS
Добавляет расширение VMAccess на виртуальную машину.

## СИНТАКСИС

```
Set-AzVMAccessExtension [-Credential <PSCredential>] [-ResourceGroupName] <String> [-VMName] <String>
 [-Name <String>] [-TypeHandlerVersion <String>] [-Location <String>] [-DisableAutoUpgradeMinorVersion]
 [-ForceRerun <String>] [-NoWait] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## ОПИСАНИЕ
Этот  cmdlet добавляет расширение VMAccess Virtual Machine Access (VMAccess) виртуальной машины VMAccess на виртуальную машину. Расширение VMAccess можно использовать для того, чтобы установить временный пароль, и он должен быть сразу же изменен после входа на компьютер. Эта поддержка не поддерживается на контроллерах доменов Windows.

## ПРИМЕРЫ

### Пример 1. Добавление расширения VMAccess
```powershell
PS C:\> Set-AzVMAccessExtension -ResourceGroupName "ResourceGroup11" -Location "Central US" -VMName "VirtualMachine07" -Name "ContosoTest" -TypeHandlerVersion "2.4" -UserName "PFuller" -Password "Password"
```

Эта команда добавляет расширение VMAccess для виртуальной машины с именем VirtualMachine07 в ResourceGroup11.
Эта команда определяет имя и версию обработвода типов для VMAccess.

## PARAMETERS

### -Credential
Указывает имя пользователя и пароль виртуальной машины в качестве **объекта PSCredential.**
Если в VM-компьютере ввести другое имя, расширение VMAccess добавит учетную запись локального администратора с этим именем и назначит ее указанному парольу. Если учетная запись локального администратора на вашем компьютере VM существует, пароль будет сброшен, а если учетная запись отключена, то расширение VMAccess включит его.
Для получения учетных данных используйте Get-Credential управления.
Для получения дополнительных сведений введите `Get-Help Get-Credential` .
