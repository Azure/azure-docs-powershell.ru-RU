# Remove-AzSqlDatabaseInstanceFailoverGroup

## КРАТКИй обзор
Удаляет группу отработки отказа экземпляра.

## Максимальное

### RemoveIFGDefault (по умолчанию)
```
Remove-AzSqlDatabaseInstanceFailoverGroup [-ResourceGroupName] <String> [-Location] <String>
 [-Name] <String> [-Force] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### Удаление группы отработки отказа экземпляра из кода ресурса
```
Remove-AzSqlDatabaseInstanceFailoverGroup [-Location] <String> [-ResourceId] <String> [-Force]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### Удаление группы отработки отказа экземпляра из определения экземпляра AzureSqlInstanceFailoverGroupModel
```
Remove-AzSqlDatabaseInstanceFailoverGroup -InputObject <AzureSqlInstanceFailoverGroupModel> [-Force]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## NОПИСАНИЕ
Эта команда удаляет группу отработки отказа для экземпляра с указанным именем, оставляя все базы данных неизменными. Конечная точка прослушивателя будет отменена из DNS.

Для выполнения команды следует использовать первичный регион для группы отработки отказа экземпляра.

## ИЛЛЮСТРИРУЮТ

### Пример 1
```
PS C:\> Get-AzSqlDatabaseInstanceFailoverGroup -ResourceGroupName rg -Location location -Name fg | Remove-AzSqlDatabaseInstanceFailoverGroup
```

Удаление группы отработки отказа для экземпляра.
