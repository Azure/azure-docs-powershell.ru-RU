# Get-AzNetworkServiceTag

## КРАТКИй обзор
Получает список ресурсов с данными тега обслуживания.

## Максимальное

```
Get-AzNetworkServiceTag -Location <String> [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## NОПИСАНИЕ
Командлет **Get-AzNetworkServiceTag** получает список ресурсов с данными об тегах обслуживания.

Обратите внимание, что указанные вами сведения о регионе Azure будут использоваться в качестве ссылки на версию (но не в качестве фильтра на основе расположения). Например, даже если вы укажете `-Location eastus2` список служебных тегов с подробными сведениями о них в разных регионах, но только в облаке, к которому относится ваша подписка (например, "общедоступный", "правительство США", "Китайский" или "Германия").

### Пример 3: получение ресурса сведений о тегах обслуживания хранилища для Западов США 2
```powershell

PS C:\> $serviceTags.Values | Where-Object { $_.Name -eq "Storage.WestUS2" }

System Service   : AzureStorage

Address Prefixes : {13.66.176.16/28, 13.66.176.48/28, 13.66.232.64/28, 13.66.232.208/28...}

PS C:\> $serviceTags.Values | Where-Object { $_.Name -like "Storage*" -and $_.Properties.Region -eq "westus2" }

Name             : Storage.WestUS2
System Service   : AzureStorage
Region           : westus2
Address Prefixes : {13.66.176.16/28, 13.66.176.48/28, 13.66.232.64/28, 13.66.232.208/28...}

Address Prefixes : {13.66.176.16/28, 13.66.176.48/28, 13.66.232.64/28, 13.66.232.208/28...}
```

Первая команда получает список ресурсов с данными тега обслуживания и сохраняет их в переменной `serviceTags` .
Следующие команды показывают различные способы фильтрации списка для выбора сведений о тегах обслуживания для хранения в Западная часть US 2.
