## ОПИСАНИЕ
Чтобы предоставить `New-AzKeyVaultRoleAssignment` доступ, используйте команду.
Access предоставляется путем назначения им соответствующей роли RBAC в нужной области.
Необходимо у указывается тема задания.
Чтобы указать пользователя, используйте параметры SignInName или Azure AD ObjectId.
Чтобы указать группу безопасности, используйте параметр Azure AD ObjectId.
Чтобы указать приложение Azure AD, используйте параметры ApplicationId или ObjectId.
Роль, которая должна быть назначена, должна быть указана с помощью параметра RoleDefinitionName pr RoleDefinitionId. Может быть указана область, для которой предоставляется доступ. По умолчанию выбранная подписка.

## ПРИМЕРЫ

### Пример 1
```powershell
PS C:\> New-AzKeyVaultRoleAssignment -HsmName myHsm -RoleDefinitionName "Managed HSM Policy Administrator" -SignInName user1@microsoft.com

RoleDefinitionName               DisplayName                    ObjectType Scope
------------------               -----------                    ---------- -----
Managed HSM Policy Administrator User 1 (user1@microsoft.com)   User       /
```

В этом примере роль "Администратор управляемой политики HSM" назначается пользователю " user1@microsoft.com в верхней области действия.
