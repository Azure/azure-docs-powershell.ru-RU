# Test-AzLogicApp

## SYNOPSIS
Проверяет определение приложения по логике.

## СИНТАКСИС

### LogicAppWithDefinitionParameterSet
```
Test-AzLogicApp -ResourceGroupName <String> -Name <String> -Location <String> [-State <String>]
 [-Definition <Object>] [-IntegrationAccountId <String>] [-Parameters <Object>] [-ParameterFilePath <String>]
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

### LogicAppWithDefinitionFileParameterSet
```
Test-AzLogicApp -ResourceGroupName <String> -Name <String> -Location <String> [-State <String>]
 [-DefinitionFilePath <String>] [-IntegrationAccountId <String>] [-Parameters <Object>]
 [-ParameterFilePath <String>] [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## ОПИСАНИЕ
С **помощью cmdlet Test-AzLogicApp** можно проверить определение приложения по логике в группе ресурсов.
Укажите имя приложения логики, имя группы ресурсов, расположение, состояние, ИД учетной записи интеграции или параметры.
Этот модуль поддерживает динамические параметры.
Чтобы использовать динамический параметр, введите его в команду.
Чтобы найти имена динамических параметров, введите дефис (-) после имени cmdlet, а затем нажимая клавишу TAB для циклическиго перебора доступных параметров.
Если параметр обязательного шаблона не задан, будет выдан запрос на его значение.

## ПРИМЕРЫ

### Пример 1. Проверка приложения логики с помощью путей к файлам
```powershell
PS C:\>Test-AzLogicApp -ResourceGroupName "ResourceGroup11" -Name "LogicApp01" -Location "westus" -State "Enabled" -DefinitionFilePath "d:\workflows\Definition.json" -ParameterFilePath "d:\workflows\Parameters.json"
```

Эта команда проверяет приложение логики с именем LogicApp01 в указанной группе ресурсов.
Команда определяет пути определения и файла параметров.

### Пример 2. Проверка приложения логики с помощью объектов
```powershell
PS C:\>Test-AzLogicApp -ResourceGroupName "ResourceGroup11" -Name "LogicApp01" -Location "westus" -State "Enabled" -Definition [IO.File]::ReadAllText("d:\Workflows\Definition.json") -Parameters @{name1="value1", name2="value2"}
```

Эта команда проверяет приложение логики с именем LogicApp01 в указанной группе ресурсов.
Команда определяет объекты определения и параметра.

### Пример 3

Проверяет определение приложения по логике. (autogenerated)
